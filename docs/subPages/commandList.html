Gitのコマンドをまとめていきます

勉強の中で出てきたコマンドを追加していきます。

VSCodeで作業しているため、基本的な管理はコマンドなくてもできますが最終的にはコマンドが必要になる。

コマンドリスト

Gitの設定に関するコマンド（初回必須）

  1. git config --list :Gitの設定を確認
  2. git config --global user.name "NAME" :ユーザ名の登録
  3. git config --global user.email "ADDRESS" ：メールアドレスの登録

Gitの操作に関するコマンド（必要最小限）

  1. git init :リポジトリの作成
  2. git status :バージョン管理する変更があるかどうか確認できる
  3. git add --all :変更をステージにすべて登録　--allでなくファイル名指定で個別登録可能
  4. git commit -m "コメント" :一行コメントをつけて履歴を登録
  5. git log --oneline :変更履歴のハッシュ値とリストとコメントを一行で表示
  6. git checkout ハッシュ値 ：ハッシュ値の示す履歴を呼び出す（その履歴に移動）
  7. git checkout master :最新のマスター履歴に戻る
  8. git reset --hard ハッシュ値 :巻き戻した履歴以降をなかったことにする

GitHubの操作に関するコマンド（必要最小限）

  1. git clone [URL]：GitHubからリポジトリをコピー（クローン：管理者は持ち主）
  2. git branch [ブランチ名]：履歴の分岐（元の履歴を汚さず作業可能）
  3. git merge [結合したいブランチ名]：今のブランチに指定したブランチを結合（git checkout master：masterに戻ってからmasterに結合が多い）
  4. git branch -d [削除したいブランチ名]：結合後、残ったブランチを削除する場合が多い
  5. git push origin：クローンもとにブランチを渡す（管理者の場合はmasterも可能だが、他社のリポジトリは別ブランチが必要）
